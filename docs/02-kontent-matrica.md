### FAQ главной (7 вопросов, драфты ответов)

1. **Сколько это стоит?» — вилка/факторы цены, ссылка на «рассчитать стоимость».
2. **Какие сроки?** — MVP за `[2–4 недели]`, полный запуск `[4–8 недель]`, зависит от объёма данных и интеграций.
3. **Кому принадлежит код?** — вам: передаём репозиторий, документацию, права.
4. **Что с безопасностью данных?** — данные не уходят третьим лицам, деплой возможен в вашу инфраструктуру, NDA.
5. **Нужно ли нам своё ПО/сервера?** — нет, поможем выбрать: облако или ваша инфраструктура.
6. **А если бот ошибётся?** — RAG отвечает по базе со ссылкой на источник; на сложном вопросе зовёт человека.
7. **С чего начать?** — заявка → бесплатный разбор задачи → предложение с оценкой.

## Как мы работаем `/process`

8 шагов, каждый — карточка: описание + артефакт («что вы получаете») + срок.

| Шаг | Артефакт | Срок (драфт) |
|---|---|---|
| 1. Заявка и бриф | Зафиксированная задача и критерии успеха | 1–2 дня |
| 2. Аудит данных и сценариев | Отчёт: какие данные есть, чего не хватает | 2–4 дня |
| 3. ТЗ и архитектура | Документ с архитектурой и планом | 3–5 дней |
| 4. Прототип / MVP | Работающий прототип на части данных | 1–3 недели |
| 5. Обучение на ваших данных (RAG) | Система отвечает по вашей базе | 1–2 недели |
| 6. Интеграции и тесты | Связка с CRM/сайтом/мессенджерами, отчёт о качестве | 1–2 недели |
| 7. Запуск | Продакшн + инструкции + код | 2–3 дня |
| 8. Поддержка и дообучение | Ежемесячный отчёт качества ответов | подписка |

Блок «что понадобится от вас»: доступы (сайт, CRM, мессенджеры), база знаний (документы, FAQ, регламенты), контактное лицо для приёмки.

## О компании `/about`

Миссия («делаем ИИ полезным, а не модным») → подход (RAG, готовые блоки, код в собственность) → чем отличаемся → команда `[фото/роли — опц.]` → стек технологий → принципы (безопасность данных, прозрачность) → счётчики `[цифры]` → CTA.

## Контакты `/contacts`

- Заголовок: «Расскажите о задаче». Подзаголовок: «Отвечаем в течение `[N]` часов».
- Форма: имя* · контакт* (Telegram / телефон / email — хотя бы одно) · «Что нужно» (select: 6 решений + «Другое / не знаю») · сообщение · чекбокс согласия на ПД*.
- Состояния: валидация инлайн · загрузка · экран «Спасибо! Ответим в течение [N] часов» с ссылками на блог/кейсы.
- Альтернативы: Telegram `[@…]`, email `[…]`, `[город — опц.]`.

## Служебные

- **/404:** «Такой страницы нет — но есть 6 решений, которые работают». Ссылки: главная, решения, блог.
- **/privacy:** политика ПД `[юрданные заказчика]` + описание cookie.
- **Пустые состояния:** блог без результатов поиска («Ничего не нашлось — посмотрите рубрики»), кейсы с пустым фильтром.
- **Footer:** описание в 1 строку + колонки (Решения ×6 / Компания: процесс, кейсы, о нас, контакты / Ресурсы: блог, рубрики, политика) + email-форма «раз в месяц — дайджест о практическом ИИ» + соцсети/Telegram + копирайт.

## Чат-виджет (тексты)

- Приветствие: «Привет! Я ассистент Aivo — тот самый продукт, который мы внедряем. Спросите про решения, цены или сроки».
- Эскалация: «Хороший вопрос — позову коллегу. Оставьте контакт, ответим в течение [N] часов».
- Fallback (если ассистент не готов к запуску): виджет с формой «задать вопрос» вместо диалога.
