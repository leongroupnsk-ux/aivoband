# Чек-лист доработки aivo.band

Приоритеты: **П1** — до активного трафика · **П2** — ближайшие итерации · **П3** — по мере роста.
Статус фиксировать здесь же (меняем [ ] на [x]).

## 1. Панель администрирования (П1) — новый блок работ

Сейчас заявки улетают только в Telegram (api/lead), контент правится в коде.
Цель: админка на `/admin` за паролем, без пересборки сайта для рутины.

- [x] **Хранилище заявок**: файловое (JSON на томе aivo-data) вместо SQLite —
      без native-зависимостей, безопаснее для сборки в Alpine. api/lead пишет
      в хранилище **и** дублирует в Telegram. Миграция на SQLite — при росте объёма
- [x] **Авторизация**: один админ (ADMIN_PASSWORD в env), сессия — подписанная
      httpOnly-кука (HMAC, ADMIN_SECRET), rate-limit на вход
- [x] **Раздел «Заявки»** `/admin`: список (имя, контакт, сообщение, дата, источник,
      решение), статусы «новая → в работе → закрыта», заметка менеджера, фильтры
- [x] Мини-дашборд: всего / новых / за неделю
- [x] Бэкап тома в крон (deploy/backup-data.sh, ротация 30 копий)
- [x] **Раздел «Кейсы и решения»** `/admin/content`: правка через JSON-редактор с
      валидацией. Кейсы — добавление/удаление/правка; решения — правка контента
      (набор из 6 зафиксирован). Данные в override-JSON на томе, страницы кейсов/
      решений стали динамическими → правки видны без пересборки. Кнопка «Сбросить».
      Добавлено поле price у решений (убрало плейсхолдер «от [X] ₽»).
      Компромисс: name решения в меню обновляется после пересборки (nav на дефолтах)
- [x] **Раздел «Блог»** `/admin/blog`: создание/правка/удаление статей из админки.
      Рантайм-статьи хранятся на томе (blog.json), рендерятся через MDXRemote в рантайме;
      страницы блога стали динамическими. Форма: заголовок, slug, дата, описание,
      категория, автор, решение-CTA, Markdown-тело, черновик. Валидация компиляцией MDX
      (битую статью не сохранить), черновики скрыты, slug демо-статей защищён.
      Файловые демо-статьи из репо продолжают работать (content/ копируется в образ)
- [x] **Пер-полевые формы для кейсов**: обычные поля вместо JSON (заголовок, метрика,
      ниша, решение-select, slug, три текстовых блока, технологии через запятую,
      результаты «было → стало» строками, флаг NDA). Решения остались на JSON —
      структура сложная (выгоды, состав, этапы), правятся редко
- [ ] Конверсия по страницам-источникам (когда будет Метрика)

## 5. Техническое (П2)

- [x] **Автодеплой** (deploy/auto-deploy.sh): сервер сам опрашивает GitHub API раз в
      5 минут и пересобирается при новом коммите. Опрос, а не GitHub Actions по SSH —
      гео-фильтр режет входящие из-за рубежа, а раннеры GitHub американские.
      Блокировка от параллельных запусков (flock + запасной вариант на mkdir),
      health-check после сборки, при ошибке sha не обновляется и прод не трогается.
      Установка — см. шапку скрипта

- [ ] Мониторинг: UptimeRobot на https://aivo.band (+ алерт в Telegram)
- [x] Страница 404 — в стиле сайта, добавлена ссылка на разборы по отраслям
- [x] Проверена нагрузка NeuroCanvas на мобильных: число точек считается от площади
      (на телефоне ~22 против ~60 на десктопе), есть пауза вне вьюпорта, отключение
      при reduced-motion и старт после простоя главного потока — оптимизировать нечего.
      **Найден и починен баг:** при ресайзе (поворот экрана) элемент мог отдать нулевую
      ширину, холст оставался нулевым и сцена в hero молча пропадала навсегда
- [ ] Замер Lighthouse на проде — из этой среды не сделать (dev-сервер нерепрезентативен).
      PageSpeed Insights тоже не сработает: он ходит с серверов Google, а их режет
      гео-фильтр. Пока фильтр не снят — мерить только локально в DevTools браузера
- [x] Заголовки безопасности (через next.config): X-Content-Type-Options, X-Frame-Options,
      Referrer-Policy, Permissions-Policy, X-DNS-Prefetch-Control. HSTS — на nginx.
- [x] **CSP** настроен с allowlist (aivochat.ru, mc.yandex.ru, googletagmanager,
      google-analytics). Проверено: посторонний скрипт блокируется, все свои грузятся.
      Тестом вскрылось, что вебвизор Метрики ходит по WebSocket — добавлен wss://,
      иначе запись сессий молча не работала бы. script-src с 'unsafe-inline'
      (у нас инлайн-инициализация счётчиков; nonce требует middleware)
- [ ] Бэкапы сервера (снапшоты Timeweb или крон-архив /opt/aivo)
