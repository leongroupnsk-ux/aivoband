## 1. Панель администрирования (П1) — новый блок работ

Сейчас заявки улетают только в Telegram (api/lead), контент правится в коде.
Цель: админка на `/admin` за паролем, без пересборки сайта для рутины.

- [ ] **Хранилище заявок**: SQLite на сервере (как в Aivo Chat), api/lead пишет заявку
      в базу **и** дублирует в Telegram (как сейчас)
- [ ] **Авторизация**: один админ-аккаунт, пароль в env, сессия в httpOnly-cookie
- [ ] **Раздел «Заявки»**: список (имя, контакт, сообщение, дата, источник-страница),
      статусы «новая → в работе → закрыта», заметка менеджера
- [ ] **Раздел «Блог»**: список статей из `content/blog`, создание/правка MDX прямо
      из админки (заголовок, описание, категория, текст), кнопка «Опубликовать»
- [ ] **Раздел «Кейсы и решения»**: правка `data/cases.ts` / `data/solutions.ts`
      через формы (перевести данные в JSON/SQLite, чтобы не пересобирать сайт)
- [ ] **Дашборд**: заявки за неделю/месяц, конверсия по страницам-источникам
- [ ] Бэкап SQLite в крон (по образцу aivo-chat/deploy/backup.sh)
